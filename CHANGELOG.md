# Changelog

## [2026-03-11] - `unreleased`

### Fixed
- 修复公司内网分流失败，企业流量改为复用系统路由和现有公司 VPN，不再依赖公司代理端口直连探测。
- 修复 macOS 上公司域名在 Go 直连失败时无法继续访问的问题，增加基于 `dscacheutil` 的系统 DNS 回退解析与按 IP 重试。
- 修复公司 NGN 与系统代理冲突导致的内网访问失败，在个人代理占用系统代理时自动切换到 TUN 共存模式。
- 修复 macOS 上点击“启动隔离”时 TUN helper 因权限不足提前退出的问题，改为通过管理员授权启动并显式返回权限错误。
- 修复个人代理端口不可达时预检直接阻止启动的问题，将其从 fail 降级为 warn，启动时再做硬检查；修复"修复网络"提权后仍显示异常的循环问题。

### Changed
- 调整预检与启动逻辑，进入 TUN 模式后会清理当前系统代理，避免 `HTTP/HTTPS/SOCKS` 代理继续干扰公司内网。
- 调整运行时恢复策略，应用初始化、预检和启动前都会尝试自动恢复残留的系统代理、DNS 和 TUN 状态。
- 放宽配置校验，系统路由模式下不再强制要求填写公司代理 Host/Port，仅校验个人代理配置。

### Added
- 新增公司流量系统路由拨号实现，以及 macOS 平台的系统路由健康探测与系统解析结果回退能力。
- 新增 TUN helper 提权启动、输出采集、后台存活检测和提权停止链路。
- 新增公司流量直连、系统路由回退、自动恢复残留网络状态和系统代理冲突下 TUN 回退的测试覆盖。
